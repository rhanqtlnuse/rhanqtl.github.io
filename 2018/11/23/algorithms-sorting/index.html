选择排序

选择最小的元素：可以使用分治的方式，将数组一分为二，递归地找出最小值。

其中有一个技巧，即选择最小值的过程中，被一分为二的数组只有一边能产生当前的最小值，而另一边的最小值其实就是整个数组中第二小的元素，下一次在获取最小值时可以直接用。

引申出算法优化的一个技巧：在算法执行的过程中，后面的计算可能会用到前面的结果，可以将前面的结果保存下来提高效率。

基于比较的排序算法时间复杂度的下界

决策树

决策树是一个二叉树，

  • 每个内结点都是一次比较操作，根据比较的结果来决定下一步的方向
  • 每个叶子结点都是输入的一种排列

对于大小为 n 的输入，共有 n! 种排列，高度为 h 的二叉树最多有 $2^h$ 个叶子结点，那么我们有：$n! \le 2^h$，即：$h \ge \log_2(n!) = \log_2((\dfrac{n}{e})^n) = \Theta(n\log n)$

也就是说，基于比较的排序算法的时间复杂度不会低于 $n\log n$，有些基于比较的排序算法的渐进复杂度为 $O(n^2)$ 是因为重复了决策树中的某些路径。

归并排序具有最优的渐进复杂度（上界和下界都为 $n\log n$）。
