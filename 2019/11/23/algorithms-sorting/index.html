摘要

0    度量标准

对于排序算法而言，最重要的度量标准是效率（性能），其次也需要考虑稳定性问题。所谓“稳定性”，就是指两个在某种规定下相等的元素在排序后保持排序前的相对位置，比如序列 $[ \dots , 1_a, 1_b, \dots ]$，如果排序后相对位置没有改变，则称排序算法为稳定的，否则称为不稳定的。稳定性在多键排序时很有用处。

1    解决问题的思路

  • 迭代：冒泡排序
  • 分治
    • 分为 1 和 n - 1
      • 选择排序
      • 插入排序
    • 分为两个大小基本相等的部分
      • 归并排序
      • 快速排序
  • 其他思路：基数排序

3.1    选择排序

有 8 种基本的选择方式：{ 从前向后，从后向前 } × { 选择最小值，选择最大值 } × { 严格不等，不严格不等 }

  • 从前向后，选择最小值：不稳定
    • 比如序列 $[100_a, 100_b, 2]$，最小值为 2，与 $100_a$ 交换之后得到的序列为 $[2, 100_b, 100_a]$
  • 从前向后，选择最大值：不稳定
  • 从后向前，选择最小值：
  • 从后向前，选择最大值，严格不等：稳定

选择最小的元素：可以使用分治的方式，将数组一分为二，递归地找出最小值。

其中有一个技巧，即选择最小值的过程中，被一分为二的数组只有一边能产生当前的最小值，而另一边的最小值其实就是整个数组中第二小的元素，下一次在获取最小值时可以直接用。

引申出算法优化的一个技巧：在算法执行的过程中，后面的计算可能会用到前面的结果，可以将前面的结果保存下来提高效率。

冒泡排序也是一种选择排序。

虽然选择排序是 $\Theta(n^2)$，与冒泡排序在最坏情况下相同，但是，选择排序在每一趟迭代中总是只需要 1 次交换，大大减少了交换的次数

4 基于比较的排序算法时间复杂度的下界

4.1 比较判别树

决策树是一个二叉树，

  • 每个内结点都是一次比较操作，根据比较的结果来决定下一步的方向
  • 每个叶子结点都是输入的一种排列

对于大小为 n 的输入，共有 n! 种排列，高度为 h 的二叉树最多有 $2^h$ 个叶子结点，那么我们有：$n! \le 2^h$，即：$h \ge \log_2(n!) = \log_2((\dfrac{n}{e})^n) = \Theta(n\log n)$

也就是说，基于比较的排序算法的时间复杂度不会低于 $n\log n$，有些基于比较的排序算法的渐进复杂度为 $O(n^2)$ 是因为重复了决策树中的某些路径。

归并排序具有最优的渐进复杂度（上界和下界都为 $n\log n$）。
