算法设计与分析：算法设计基础

发表于 2018-12-04 | 分类于 Algorithms

分治

框架：

  • 将问题分解为较小的子问题
    • 有些情况下，可能只要解决了问题的一部分就能得到问题的答案，这种情况叫做“规约”
  • 解决子问题
    • 基本情况可以直接解决
    • 非基本情况可以继续分解成更小的子问题
  • 将子问题的结果合并

算法设计过程中，一个不错的思路是先设计出相应的递归算法，然后对其进行优化或者转换成迭代形式。

递归算法设计

设计递归算法有两个必须遵守的基本原则：

  • 必须有基本情况
  • 每次分解/规约问题要确保至少有一个参数确定能够更加接近结束条件

例子：扔鸡蛋

假设我们有两个鸡蛋，要确定他们的强度（两个鸡蛋强度相同）。强度定义如下：

  • 如果从第 i 层楼将鸡蛋扔下，鸡蛋碎了，那么鸡蛋的强度为 i - 1

如果我们有两个鸡蛋，强度在 1 到 100 之间，如何确定鸡蛋的强度？

解决方案

选择第 i 层将鸡蛋扔下，

  • 如果鸡蛋碎了，那么可以确定强度在 1 到 i - 1 之间，用另一个鸡蛋从第 2 层楼开始顺序查找；
  • 如果鸡蛋没碎，那么可以确定强度在 i 到 100 之间，此时我们还有 2 个鸡蛋，可以再次进行整个过程。

可以看到，这个问题有两部分参数：

  • 鸡蛋的个数
  • 当前已确定的强度范围

下面，我们对这个问题进行推广。

推广

如果我们有 k 个鸡蛋，要在 m 到 n 之间确定一个强度，应该怎样做才能有最小的次数？

解决方案

选择第 i 层（m < i <= n），扔下一个鸡蛋，

  • 如果鸡蛋碎了，那么可以确定强度在 m 到 i - 1 之间，此时我们还有 k - 1 个鸡蛋
  • 如果鸡蛋没碎，那么可以确定强度在 i 到 n 之间，此时我们还有 k 个鸡蛋

算法的优化

算法优化有一些常见的技巧：

  • 以空间换时间：我们的算法可能会出现后面的计算要用到前面计算结果的情况，未经优化的算法可能会进行大量的重复计算，这时候我们可以将这些中间结果记录下来，用到的时候直接查询，无需计算。

  • 将递归转换为迭代

    • 尾递归：在运算的最后进行的递归调用

      • 是尾递归

      • 不是尾递归

        注意尾递归是在运算的最后出现的递归调用，而在上面例子中，最后的运算是加法

    • 尾递归可以方便地转换成迭代

      • 函数体转换为一个循环体

      • 参数转为变量声明

      • 基本情况：直接返回相应的结果

      • 递归调用：首先计算相应的新参数，赋给相关变量，然后加上 continue 语句

    • 其它形式的递归也可以转换成迭代

      • 斐波那契数列：计算 f(n) 首先要顺序地计算 f(0)，f(1)，· · ·，f(n - 1)

  • 某些问题还有特定的优化方式（根据计算过程优化）

扔鸡蛋的优化

扔鸡蛋问题可以根据下面几点来优化：

  • 假设初始给定的 k’，m’，n’，那么对于参数 k <= k’，m <= m’，n <= n’，函数 f(k, m, n) 可能会被多次重复计算
  • 对于任意的 k，m，n，m’，n’，如果 n - m = n’ - m’，那么 f(k, m, n) = f(k, m’, n’)
  • 根据题意
    • k’ > k’’ 意味着 f(k’, m, n) <= f(k’’, m, n)
    • m - n > m’ – n’ 意味着 f(k, m, n) >= f(k, m’, n’)

应对方案：

  • 重复计算：以空间换时间

  • 第二点表明，只要 m 与 n 的差相同，计算的结果就相同，所以我们可以只保留二者之差 n - m，这样能够减少一个参数（因为某一个参数总为 0）

  • 第三点涉及到函数的单调性问题

    • f 对于 k 是一个单调非递增函数
    • f 对于 m 与 n 的差是一个单调非递减函数
    • 根据上面两点，可以确定，两条曲线一定会交于某点，并且 f 在这一点取得最小值

什么是红黑树？

红黑树首先是一棵二叉搜索树，具有几乎完美的平衡性质

叙述性定义

红黑树是这样一棵二叉搜索树：

  • 根结点为黑色
  • 每个内部结点要么是黑色，要么是红色
  • 叶结点（NIL）为黑色
  • 颜色约束：红色结点不能有红色子结点（换言之，红色结点只能有黑色子结点）
  • 黑色高度约束：从某结点出发到其所有后代叶结点的简单路径上，均包含相同数目的黑色结点

递归定义

  • ARB（Almost-RB）树：除根结点为红色之外，满足 RB 树其他约束
  • 定义 RB$_0$ 为空树（或外部结点），以 $h$ 表示树的黑高度
  • RB$_{h+1}$：子树可以是 RB$_h$ 或 ARB$_{h+1}$
  • ARB$_{h+1}$：子树均为 RB$_h$

为什么要使用红黑树？

C++ 中的 map

我们知道 HashMap 的平均时间复杂度为 O(1)（可以参考《算法导论》的相关分析），map 的平均时间复杂度为 $O(\lg n)$ 的，实践上是不是 HashMap 一定优于 map 呢？这里面有几个因素要考虑：

  1. HashMap 内存效率比 map 差，这是显而易见的；
  2. map 的查找效率实践上是非常高的，如在 1M 数据中查找一个元素，仅需要 20次 比较；
  3. map 的查找效率比 HashMap 稳定，hash 最坏的情况为 $O(n)$（个人观点：无论使用什么 hash 函数，总能够找到最坏情况的键序列，虽然可以使用全域哈希的方法，但也并不是完全能够避免最坏情况，而且 hash 函数一不小心就会…，相比之下，红黑树的 $O(\lg n)$ 很稳，而且在可以接受的范围内）；
  4. HashMap 查找时候要算 hash，这个最坏时间复杂度是 $O(m)$（M是key字符串的长度），如果你的 key 非常非常非常非常非常非常……长，基于比较的 map 通常只使用头几个字符进行比较，而 HashMap 要 $O(M)$ 地算出 hash；
  5. 内存布局会影响内存局部性，对性能会有影响。

二分搜索

如果函数 f(x) 是单调的，并且在已知区间 [a, b] 中存在某个值 c 使得 f(c) 等于给定的值 d，那么可以使用二分法来求出 c。

红黑树

（注意高度与黑高度的不同！）

定义

一个二叉搜索树，每个节点或为红色，或为黑色（所有的外部节点都为黑色），满足如下约束：

  • 颜色约束：红色的结点不能有红色的子结点
  • 黑高度约束：给定节点 u 的所有外部路径的黑色长度都相同
  • 根结点是黑色的

ARB：根结点为红色，满足颜色约束和黑高度约束

递归定义

  • 基本情况：RB$_0$ 为外部结点
  • ARB$_h$（h >= 1）
    • 根节点为红色
    • 两棵子树都为 RB$_{h-1}$
  • RB$_h$（h >= 1）
    • 根结点为黑色
    • 子树可以是 RB$_{h-1}$ 或 ARB$_h$

性质

  • RB$_h$ 或 ARB$_h$ 的黑高度是良定义的，为 h
    • “良定义” 是指根结点的所有外部路径的黑色长度都相同
  • RB$_h$
    • 至少有 2$^h$ - 1 个黑色内部结点
    • 至多有 4$^h$ - 1 个内部结点
    • 任何一个黑色结点的深度至多是树的黑色深度的两倍
  • ARB$_h$
    • 至少有 2$^h$ - 2 个黑色内部结点
    • 至多有 4$^h$ / 2 - 1 个内部结点
    • 任何一个黑色结点的深度至多是树的黑色深度的两倍
  • 有 n 个内部结点的红黑树的黑色深度至多为 log(n+1)，树的高度至多为 2 log(n+1)

操作

插入

思路：类似二叉搜索树，保持黑高度约束（将新插入的结点设为红色），调整树以符合颜色约束

四种情况

插入之后可能有两种情况：

  • 插入之后的子树既不是 RB 也不是 ARB
  • 插入之后的子树是 ARB，但树的根结点是红色的

下面用五个常量来表示插入之后的状态。

插入之后树的顶端颜色状态
  • ok：不需要调整
  • rbr：插入之后的树为 RB
  • brb：插入之后的树为 ARB，如果父结点的颜色为红色，需要调整（此时从父结点的角度来看，树的顶端颜色状态为 rrb 或 brr）
  • rrb：插入之后的树的根结点为红色，左子树是 ARB，右子树是 RB（此时父结点一定是黑色的）。要根据父结点的情况来确定是 3-结点簇还是 4-结点簇
  • brr：插入之后的树的根结点为红色，左子树是 RB，右子树是 ARB。调整方法与 rrb 类似

删除

思路：类似二叉搜索树，保持颜色约束，调整树以符合黑高度约束

  • 如果删除之后，黑高度不变，不需要调整，删除完成
  • 如果删除之后，子树的黑高度减 1，需要调整

基于比较的排序算法时间复杂度的下界

决策树

决策树是一个二叉树，

  • 每个内结点都是一次比较操作，根据比较的结果来决定下一步的方向
  • 每个叶子结点都是输入的一种排列

对于大小为 n 的输入，共有 n! 种排列，高度为 h 的二叉树最多有 $2^h$ 个叶子结点，那么我们有：$n! \le 2^h$，即：$h \ge \log_2(n!) = \log_2((\dfrac{n}{e})^n) = \Theta(n\log n)$

也就是说，基于比较的排序算法的时间复杂度不会低于 $n\log n$，有些基于比较的排序算法的渐进复杂度为 $O(n^2)$ 是因为重复了决策树中的某些路径。

归并排序具有最优的渐进复杂度（上界和下界都为 $n\log n$）。

有向无环图（DAG）

一个有向无环图一定至少有一个顶点没有入边，有一个顶点没有出边

判断有向图环路的方法：如果有回边（Back Edge），那么一定有环路

拓扑排序

一个 DAG 可以有多个不同的拓扑排序。

有两种思路：

  • 将当前顶点排好序，再对后代进行排序
    • 利用入度（BFS）
      • 快速找到入度为 0 的顶点：队列，当某个顶点入度为 0 时，将其入队
      • 当某个顶点的入度减到 0 之后，就不会再减小
  • 递归地将后代排好序，然后当前顶点自然就排好了（DFS）

但是有一个问题：我们必须首先找到 C3 的 Leader，而这个问题没办法解决（要么从头（C1）的 Leader 开始遍历，要么随机选择一个顶点，但是这两种方式都没办法确定一个顶点是不是 C3 的 Leader）。

所以我们还可以这样想：先找出 C1 中的所有顶点，然后是 C2/C4，最后是 C3。那么问题来了，由于强连通分量之间存在边，所以访问并不一定是先访问完某一个强连通分量中的顶点，然后才去访问其他强连通分量中的顶点，而很有可能是访问了当前强连通分量中的部分顶点之后，转而访问其他强连通分量。

注意到一个问题：不管是发现顺序还是结束顺序， C3/C4 的 Leader 和全部顶点都是一起出现的，将 C3 和 C4 拿掉之后，C2 的 Leader 和全部顶点又出现在了一起，再将 C2 拿掉，最终 C1 的 Leader 和全部顶点也出现在了一起。

注意到不管是发现顺序还是结束顺序，每个强连通分量中的顶点总数是相同的，这意味着我们可以用一个容器将发现顺序保存起来，每当我们完成一个 Leader 的访问时（这意味着这个 Leader 对应的强连通分量已经完成了），就开始从容器中删除顶点，直到将容器中的相同的 Leader 删除，这时候我们就找到了一个强连通分量。

确定 Leader

有了上面的算法，我们还需要解决一个很早就提出的问题：如何确定一个顶点是 Leader。

再来看我们的抽象强连通分量图：

每一条边所指的顶点就是对应强连通分量的 Leader。很容易发现一个问题：两个存在边的强连通分量之间只有单向的路径。也就是说，一个强连通分量内部可能有很多回边（Back Edge），但是绝不会指向遍历树中比 Leader 更高层的顶点。这就是我们确定 Leader 的依据。
